================================================================================
               ПОВНИЙ КАТАЛОГ ФУНКЦІОНАЛУ — WoW RAID MANAGEMENT PLATFORM
================================================================================

Платформа для управління рейд-гільдіями у World of Warcraft з інтеграцією
Blizzard API, Warcraft Logs, Raider.io, Discord та AI-аналітикою (Gemini).


================================================================================
  1. АВТЕНТИФІКАЦІЯ ТА УПРАВЛІННЯ АКАУНТОМ
================================================================================

1.1  Вхід через Battle.net OAuth
     - Авторизація через офіційний Blizzard OAuth2 (wow.profile scope)
     - Автоматичне створення або оновлення акаунту при вході
     - Обробка rate-limiting з exponential backoff

1.2  Прив'язка Discord акаунту
     - OAuth-авторизація через Discord
     - Зберігання Discord ID та username
     - Відв'язка Discord у будь-який момент

1.3  Управління профілем
     - Видалення акаунту з повним очищенням даних
     - Вибір мови інтерфейсу (English / Українська)

1.4  Ролі та дозволи
     - Owner (Власник статік-групи) — повний контроль
     - Officer (Офіцер) — управління рейдами, розкладом, казною
     - Member (Учасник) — RSVP, перегляд даних
     - Окремі гейти: manage, canAccessSettings, canManageTreasury,
       canManageSchedule, canAnnounceToDiscord, canViewGlobalReport


================================================================================
  2. СТАТІК-ГРУПИ (ГІЛЬДІЇ)
================================================================================

2.1  Створення статік-групи
     - Створення нової групи вручну (назва, реалм, регіон)
     - Автогенерація інвайт-коду

2.2  Запрошення та приєднання
     - Генерація інвайт-посилань (secure token)
     - Публічна landing-сторінка інвайту (без авторизації)
     - Приєднання через інвайт-токен
     - Persistent join flow — збереження інвайту через Battle.net редирект

2.3  Онбординг (покроковий майстер)
     - Прогрес-бар з кроками та іконками
     - Крок 1: Вибір — створити або приєднатися
     - Крок 2: Створення / Приєднання через код
     - Крок 3: Синхронізація персонажів з Battle.net
     - Крок 4: Вибір основного та рейд-персонажів, призначення спеків
     - Контекстна довідка на кожному кроці

2.4  Управління учасниками
     - Перегляд списку учасників з їх персонажами
     - Зміна ролей учасників (Officer / Member)
     - Зміна статусу (Active / Benched / Reserve)
     - Видалення учасників зі статік-групи
     - Передача власності іншому учаснику


================================================================================
  3. ПЕРСОНАЖІ
================================================================================

3.1  Синхронізація з Blizzard API
     - Імпорт усіх персонажів акаунту з Battle.net
     - Синхронізація профілю: ilvl, active spec, аватар
     - Синхронізація екіпірування (по кожному спеку окремо)
     - Mythic+ рейтинг та пройдені ключі
     - Рейд-прогресія (вбивства босів)
     - Achivements та статистика
     - PvP дані (2v2, 3v3, RBG, Shuffle рейтинги)
     - Завершені квести (тижневі, Prey)
     - Репутації фракцій
     - Колекція петів (кількість унікальних, рівень 25)
     - Delve tier completion

3.2  Управління персонажами
     - Вибір основного персонажа (main)
     - Вибір рейд-персонажів (checkboxes)
     - Таблиця з ilvl, реалмом, статусами
     - Автозбереження змін у реальному часі

3.3  Спеціалізації
     - Вибір доступних спеків для кожного персонажа у статіку
     - Призначення main spec (жовта зірка)
     - Піктограми спеків з ролями (Tank/Heal/Melee/Ranged)
     - Мульти-вибір спеціалізацій

3.4  Raider.io інтеграція
     - Автоматичний fetch профілю з Raider.io
     - M+ score, рейтинги, рейд-прогресія, gear, talents


================================================================================
  4. ДАШБОРД (ТАКТИЧНИЙ ОГЛЯД)
================================================================================

4.1  Картка досягнень статіку
     - Таблиця прогресії рейд-босів по вінгам
     - Складність досягнення кожного боса
     - Hover-тултіпи з датами прогресії

4.2  Готовність ростеру
     - Прогрес-бари по ролям: Tank / Healer / Melee / Ranged
     - Поточна кількість vs максимальна місткість
     - Кольорове кодування

4.3  Наступний рейд
     - Живий зворотній відлік до наступного рейду
     - Бейджі статусу (Planned / Discord Posted)
     - Доступні ролі
     - Кнопка швидкого RSVP

4.4  Логістика
     - Резерви гільдійського банку (золото)
     - Тижні автономності (скільки тижнів вистачить казни)
     - Прогрес збору тижневих внесків (10-слотова сітка)
     - Вартість витратних матеріалів на рейд-день

4.5  Майбутні події
     - Скролюваний список запланованих рейдів
     - Дата/час, кількість RSVP, швидкий перехід

4.6  Статус синхронізації
     - Круглі індикатори для трьох сервісів (Blizzard / Raider.io / WCL)
     - Таймери зворотнього відліку до наступної синхронізації
     - Детальний модал стану синхронізації


================================================================================
  5. РЕЙД-КАЛЕНДАР ТА РОЗКЛАД
================================================================================

5.1  Місячний календар
     - Вигляд по місяцях з навігацією (Назад / Вперед / Сьогодні)
     - 7-колонкова сітка (Пн-Нд)
     - Дні з подіями виділені, кількість подій
     - Клік по дню — створення нової події
     - Клік по події — редагування

5.2  Створення рейд-подій
     - Вибір таймзони
     - Каруселі вибору часу (start/end) в стилі iOS
     - Індикатор overnight-євенту (якщо закінчується наступного дня)
     - Поле опису
     - Перевірка дублікатів (той самий день)

5.3  Автогенерація подій
     - Автоматичне створення рейдів на 30 днів вперед
     - Базується на налаштованому розкладі (дні + час)
     - Врахування таймзон


================================================================================
  6. РЕЙД-МЕНЕДЖМЕНТ ТА ПОДІЇ
================================================================================

6.1  Сторінка рейд-події
     - Хедер з датою/часом, складністю, статусом
     - Лічильники ролей (Tanks/Healers/Melee/Ranged — поточні/ліміт)
     - Відображення поточного персонажа користувача
     - Кнопки: RSVP / Редагувати / Видалити / Announce в Discord

6.2  RSVP система
     - Вибір персонажа для рейду (dropdown з класом)
     - Вибір спеціалізації (3-колонкова сітка з роль-бейджами)
     - Статус присутності: Present / Absent / Tentative / Late
     - Коментар до RSVP
     - Блокування RSVP після старту рейду
     - Свіч персонажа / спеку після RSVP

6.3  Ростер рейду
     - Повний grid по ролям (Tank/Heal/Melee/Ranged)
     - Картки персонажів: аватар, ім'я (у кольорі класу), спек, статус
     - Індикатори статусу (зелений/жовтий/червоний/білий)
     - Іконка коментаря при наявності нотатки
     - Absent/Tentative виділені окремо під роздільником

6.4  Компактний ростер
     - Кнопки імен гравців з іконками спеків та статус-точками
     - Lockout ring indicator (жовта рамка якщо бос вже вбитий)
     - Bench/Unbench на hover
     - Split-групи (A/B/C/D) з dropdown для менеджерів
     - Підсвічення гравців, яких часто бенчать

6.5  Encounter Sidebar (Бос-панель)
     - "All Encounters" з мульти-чекбоксом
     - Групування по вінгам інстансу
     - Кнопки босів з портретами
     - Лічильник ростеру (selected/inherited/total)
     - Lock-іконка для менеджерів
     - Encounter-specific ростер tracking

6.6  Encounter Roster (Ростер під кожного боса)
     - Призначення персонажів на конкретних босів
     - Статуси: selected / queued / benched
     - Масове оновлення ростеру під боса
     - Статистика: % вибору для кожного персонажа

6.7  Buff/Utility Checklist
     - Баффи/дебаффи: наявність (зелена галочка / червоний хрест)
     - Утиліти: кількість провайдерів по класах
     - % покриття
     - Виділення відсутніх баффів

6.8  Composition Warnings
     - Нестача ролей (червоні помилки)
     - Дисбаланс цілителів (жовті попередження)
     - Переповнення melee
     - Відсутність критичних баффів (Bloodlust, Combat Res)
     - Часто бенчені гравці (синя інформація)

6.9  Override та Split-менеджмент
     - Override attendance status (тільки рейд-лідер)
     - Розподіл по split-групам (1-4 спліти)
     - Призначення boss-планів до encounter-ів


================================================================================
  7. BOSS PLANNER (СТРАТЕГІЧНЕ ПЛАНУВАННЯ)
================================================================================

7.1  Повноекранний редактор стратегій
     - Навігатор по фазам/кроках (Previous / Next / Save)
     - Групування босів по вінгам інстансу
     - Підтримка складностей: Normal / Heroic / Mythic

7.2  Інтерактивна карта (SVG Canvas)
     - Zoom/Pan з колесом миші та middle-mouse
     - Координатна сітка (960x540)
     - Індикатор рівня зуму (%)
     - Edge auto-scroll при перетягуванні
     - Гравці-токени, маркери, фігури, текст, waypoints
     - Індикатори напрямку для гравців/груп
     - Обертання елементів
     - Клавіатурні скорочення

7.3  Панель інструментів
     - Інструменти: Select / Draw / Marker / Shape / Label / Text / Waypoint
     - Рейд-маркери: Skull, X, Focus тощо
     - Фігури: Circle, Rectangle, Arrow, Line, Cone, Text
     - Палітра кольорів
     - Toggle заливки
     - Undo / Redo
     - Очищення канвасу

7.4  Палітра гравців
     - Картки гравців з main-персонажем
     - Drag-drop призначення на спліти
     - Розбивка по ролях
     - Підсумки по сплітах

7.5  Шерінг планів
     - Генерація shareable-посилань (binary hex token)
     - Публічний перегляд плану (без авторизації)
     - Відкликання/видалення посилань

  8. РОСТЕР ТА АУДИТ ЕКІПІРУВАННЯ
================================================================================

8.1  Уніфікований ростер
     - Селектор тижнів (Live Week + історичні снепшоти)
     - Табова система: Summary / Gear / Raids / Vault
     - Сортовані колонки
     - Кешування тижневих снепшотів
     - Дані по main + альтам

8.2  Summary Tab
     - Загальний огляд ростеру
     - Основні метрики персонажів

8.3  Gear Tab — Аудит екіпірування
     - Виявлення відсутніх зачарувань по слотах
     - Виявлення некоректних (низькоякісних) зачарувань
     - Аудит сокетів
     - Трекінг апгрейдів
     - Spark/Embellishment валідація
     - Ідентифікація тір-сету
     - Класифікація по ilvl-тірах

8.4  Raids Tab
     - Тижнева участь у рейдах
     - Кількість пройдених ранів

8.5  Vault Tab — Great Vault
     - M+ vault: враховані ключі (keystone + regular + heroic mythic)
     - Raid vault: вбиті боси
     - Ilvl-таблиці нагород vault
     - Dungeon vault lookup

8.6  Прогресія статіку
     - Автоматичний перерахунок на основі 60% ростеру
     - Трекінг нових вбивств босів з таймстемпами
     - Усі складності: LFR / Normal / Heroic / Mythic
     - Захист від даунгрейду

  9. КАЗНА (TREASURY)
================================================================================

9.1  Дашборд казни
     - Загальні резерви (в золоті)
     - Калькулятор тижневого податку (на гравця)
     - Фінансове здоров'я (тижні автономності)
     - Сітка прогресу збору (10 слотів: оплачено/очікується)
     - Трекінг по учасниках (Covered / Not Covered + баланс)
     - Кнопки Deposit / Withdrawal (для менеджерів)
     - Модал налаштування податку

9.2  Планувальник витратних матеріалів
     - Сітка карток рецептів з іконками
     - Поля кількості на рецепт
     - Автоматичний розрахунок:
       — Тижнева вартість матеріалів
       — Внесок з податку на крафт
       — Економія кожного рейдера
       — Множник рейд-днів
     - Автозбереження (500ms debounce)

9.3  Історія транзакцій
     - Таблиця: Дата / Учасник / Тип / Сума / Опис
     - Фільтр по учаснику
     - Бейджі типу (Deposit — зелений / Withdrawal — червоний)
     - Редагування транзакцій (менеджери)
     - Коментарі до транзакцій

9.4  Автоматичний тижневий податок
     - Автоматичне нарахування при Weekly Reset
     - Розрахунок на основі вартості витратних матеріалів


================================================================================
  10. WARCRAFT LOGS & AI-АНАЛІТИКА
================================================================================

10.1 Індекс логів
     - Фільтрація по діапазону дат (flatpickr calendar)
     - Фільтр складності (Mythic / Heroic / Normal / Raid Finder)
     - Пагінація
     - Ручне додавання логів через WCL URL
     - Кулдаун на додавання (60 хв, таймер MM:SS)
     - Захист від дублікатів

10.2 Деталі логу
     - AI-згенерований аналіз рейду
     - Метадані логу
     - Метрики ефективності гравців

10.3 AI-чат аналіст (Gemini)
     - Інтерактивні запитання щодо рейд-логів
     - Контекст на основі ролі:
       — Лідери: повний глобальний контекст
       — Учасники: тільки персональні дані
     - Історія повідомлень (останні 6 msg)
     - Персональні та командні тактичні звіти

10.4 Автоматичний аналіз
     - Пошук логів після завершення рейду (time window matching)
     - AI-аналіз через Gemini API (JSON response mode)
     - Генерація PersonalTacticalReport для кожного персонажа
     - Публікація результатів у Discord webhook

10.5 WCL інтеграція (GraphQL API v2)
     - Мульти-запит: fights/phases → tables → rankings
     - Дані: боси, фази, смерті, damage, healing, casts
     - Витратні матеріали, баффи, дебаффи, resource waste
     - Метрики: DPS/HPS, parse %, рейтинги
     - Пошук гільдії та fetch звітів

================================================================================
  13. ФОНОВА СИНХРОНІЗАЦІЯ ТА АВТОМАТИЗАЦІЯ
================================================================================

13.1 Синхронізація персонажів
     - Паралельні запити до Blizzard API (profile, equipment, M+, raid,
       achievements, quests, PvP, reputations)
     - Паралельний fetch з Raider.io
     - JSON Schema валідація кожного ендпоінту
     - Grace period для квестових даних (2 год після weekly reset)
     - Снепшоти vault по тижнях (8 тижнів ретеншн)
     - Equipment-by-spec tracking

13.2 Компіляція даних персонажів
     - Raw data → CharacterDataDTO + CharacterWeeklyDataDTO
     - Одноразовий pre-load спеків (без N+1)
     - Агрегація всіх аудит-сервісів
     - Persist у character_data + character_weekly_data

13.3 Синхронізація статіків
     - Автоматичний dispatch для всіх статіків
     - Перевірка інтервалів синхронізації по тірах
     - Мітки часу (BNET / RIO / WCL)

13.4 Аукціонний дім
     - Streaming fetch commodities з Blizzard API
     - Extraction мінімальних цін для відстежуваних предметів
     - Bulk insert price snapshots (500-item chunks)

13.5 Weekly Reset
